# 生成外键注意

## 问题

在物理模型生成创建数据库的脚本时，由于原来生成外键的规则问题，生成的外键在表名较长的情况下，有可能会重名

## 修改外键规则

### 方案一

1. 选择 `Database` > `Edit Current DBMS`
2. 选择 `Scripts` > `Objects` > `Reference` > `Create`
3. PostgreSQL
  可以发现右侧的 `Value` 为：
  
  ```
  alter table [%CQUALIFIER%]%CHILD% add [constraint %CONSTNAME% ]foreign key[%R%? %CONSTNAME%] (%FKEYCOLUMNS%)
      references [%PQUALIFIER%]%PARENT%[ (%CKEYCOLUMNS%)][ %ReferenceMatch%][ on delete %DELCONST%][ on update %UPDCONST%]
  ```

  将其改为:
  
  ```
  alter table [%CQUALIFIER%]%CHILD% add [constraint fk_%FKEYCOLUMNS%__from__%PARENT% ]foreign key[%R%? %CONSTNAME%] (%FKEYCOLUMNS%)
        references [%PQUALIFIER%]%PARENT%[ (%CKEYCOLUMNS%)][ %ReferenceMatch%][ on delete %DELCONST%][ on update %UPDCONST%]
  ```
4. MySQL
  可以发现右侧的 `Value` 为：
  
  ```
  alter table %TABLE%
   add [constraint %CONSTNAME% ]foreign key (%FKEYCOLUMNS%)
      references %PARENT%[ (%CKEYCOLUMNS%)]
     [ on delete %DELCONST%][ on update %UPDCONST%][%Deferrable%? deferrable[%ForeignKeyConstraintDeferred%? initially deferred: initially immediate]]
  ```

  将其改为:

  ```
  alter table %TABLE%
     add [constraint fk_%FKEYCOLUMNS%__from__%PARENT% ]foreign key (%FKEYCOLUMNS%)
        references %PARENT%[ (%CKEYCOLUMNS%)]
       [ on delete %DELCONST%][ on update %UPDCONST%][%Deferrable%? deferrable[%ForeignKeyConstraintDeferred%? initially deferred: initially immediate]]
  ```
5. 选择 `Scripts` > `Objects` > `Key` > `ConstName`
6. PostgreSQL
  可以发现右侧的 `Value` 为：

  ```
  AK_%.U18:AKEY%_%.U8:TABLE%
  ```

  将其改为:

  ```
  AK_%.U18:AKEY%_%.U30:TABLE%
  ```


### ~~方案二~~

选择 `Database` > `Edit Current DBMS`
选择 `Scripts` > `Objects` > `Reference` > `ConstName`
可以发现右侧的 `Value` 为：
`FK_%.U8:CHILD%_%.U9:REFR%_%.U8:PARENT%`

可见，该命名方法是：'FK_'+8位子表名+9位Reference名+8位父表名，你可以根据这种模式自定义为：
`FK_%CHILD%_AND_%PARENT%`

如果要去除前缀，也可以设置为：

`FK_%1.30M:CHILD%_AND_%1.30M:PARENT%`

**注意，如果无法保存，请使用管理员来打开PowerDesigner**
